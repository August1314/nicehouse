## 作业要求梳理（好房子三维可视化系统）

> 来源：`作业要求.md`，这里只做简要提炼，方便团队在 GitHub 上协作。

### 总体目标

- 使用 Unity 构建一个 **“好房子”三维可视化系统**，围绕 **安全、舒适、绿色、智慧** 四大核心需求，体现 **以人为中心** 的设计理念。

### 基础要求（70 分）

- **三维模型（30 分）**
  - 完成好房子三维模型的建模（或导入现有模型）
- **功能可视化（20 分）**：至少 2 项，交互性强（可自定义）
  - 环境智控
  - 能耗智控
  - 智能监护
  - 健康监测
  - 一键呼叫
  - 安全防护
- **系统分层查看（20 分）**：至少 2 项，交互性强（可自定义）
  - 剖面图
  - 温度
  - 湿度
  - PM2.5
  - 活动热力图

### 进阶要求（30 分）

- 所有功能项完整可视化，交互逻辑合理（10 分）
- 所有系统分层查看完整实现，动态展示合理（10 分）
- 构建三维数字人模型，与好房子融合，体现“以人为中心”，并加入有逻辑的特色新功能（10 分）

### 建议技术选型（本组计划使用）

- 三维引擎：**Unity**
- 编程语言：**C#**
- 数据可视化：如有需要可考虑接入图表（UI + 简单图表逻辑）
